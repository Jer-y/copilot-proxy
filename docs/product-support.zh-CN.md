# 产品支持

## 产品定义

copilot-proxy 是面向本地单用户的协议适配器，让受信任的客户端通过兼容 OpenAI 和 Anthropic 的 API 使用同一个 GitHub Copilot 身份。

同一进程会共享一个上游身份、账号路由、模型目录快照、恢复熔断器、并发边界和诊断入口。选择 `business` 或 `enterprise` 账号类型只会改变 Copilot 上游路由，不会增加下游用户或企业管控能力。

## 部署支持矩阵

| 拓扑 | 支持状态 | 产品边界 |
| --- | --- | --- |
| 一名受信任用户通过本机回环地址使用 | **支持** | 符合单身份设计，凭据和状态保留在本机 |
| 认证网关后的私有上游 | **有条件支持** | 网关和网络必须提供下游认证、授权、限额与隔离 |
| 团队直接共享监听器 | **不支持** | 代理没有下游用户认证或租户隔离 |
| 公共多租户 API | **不支持** | 代理没有公共服务安全边界、凭据隔离、分布式限流、计费、审计或高可用控制平面 |

有条件支持私有网关，不代表绑定 `0.0.0.0` 后即可安全共享。代理必须保持私有，且只能通过认证网关访问。所需边界见[部署](deployment.zh-CN.md)。

## 非目标

copilot-proxy 不用于构建：

- 公共托管推理服务；
- 用户、API Key、租户、配额、计费或审计系统；
- 高可用或分布式控制平面；
- 仅依据 OpenAI 或 Anthropic 文档判断上游能力的机制；
- 静默丢弃用户意图后仍报告成功的兼容层。

公共服务或独立多租户产品需要单独的控制平面架构，不能通过逐步暴露当前进程来实现。
